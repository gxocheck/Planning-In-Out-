🏷️ Réglages CR client

Les CR client sont rattachés au code destinataire (colonne L du fichier OUT). À chaque import, la colonne CR client se remplit automatiquement à partir de cette base. La base GXO BP destinataire est préchargée — modifie / ajoute / supprime quand un CR évolue.

Diffusion temps réel — multiviewer

localStorage est propre à chaque navigateur : un simple fichier envoyé par mail ne se synchronise pas. Pour que tes collègues / le client voient tes mises à jour en direct, branche une base Firebase (gratuite) — la config voyage ensuite avec le lien ou le fichier client.

Firebase Realtime Database

Pour partager (au choix) : ① héberge ce fichier une fois (GitHub Pages / intranet) puis « Copier le lien multiviewer » : tout le monde ouvre le même lien et voit le live (le client se connecte avec son code). ② ou « Fichier client à héberger » : copie autonome avec la config intégrée.
